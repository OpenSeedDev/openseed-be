# VS-037 경제 데이터 정합성 검사

## 시스템 결과

주기적인 읽기 전용 검사가 Wallet, Ledger, Unit Lot, Recovery와 대기 지급의 불일치를 탐지하고, 실행 결과와 개별 발견 사항을 운영자가 DB에서 추적할 수 있게 보존한다.

## 근거와 선행 조건

- 백로그: VS-037, 선행 VS-036, resource lock `unit`·`wallet`
- 긴급 배포용 Stack 근거: VS-036 PR #78 Head `9e96282fe395e5c3f54a60f24a7dd1fdecb9a18f`
- 선행 PR은 아직 Merge되지 않았으며 본 PR은 #78 브랜치를 부모로 사용한다.
- 정책: 탐지 결과만 기록하며 자동 정정하지 않는다.

## 포함 범위

- 여러 서버에서 중복 실행되지 않는 시간별 정합성 검사
- 지갑 잔액과 최신 원장 잔액, 원장 잔액 연속성 검사
- 지갑 회수 대기 잔액과 회수·지급 합계 검사
- Lot과 구매 원장, 회수 상태와 Recovery 검사
- Recovery·대기 지급과 대응 Point 원장 검사
- 검사 실행과 개별 불일치의 append-only 추적 결과 저장
- 새 UTC Flyway Migration과 ERD 갱신

## 제외 범위

- 데이터 자동 수정 또는 정정 원장 생성
- 관리자 페이지·외부 조회 API·수동 실행 API
- 경보와 운영 지표 연동: OPS-03
- 과거 가격 재평가

## 데이터 영향

- `financial_consistency_checks`는 실행 시각, 완료 시각, 상태와 발견 수를 보존한다.
- `financial_consistency_findings`는 검사별 코드, 대상 사용자·엔티티와 기대·실제 값을 보존한다.
- 두 테이블은 운영 추적을 위해 UPDATE·DELETE를 거부하고 정리 시 TRUNCATE만 허용한다.

## 인수 조건과 테스트 목록

- [ ] 정상 구매·회수·대기 지급 데이터는 `CONSISTENT`, 발견 0건으로 기록한다.
- [ ] 지갑 잔액과 최신 원장 잔액 불일치를 탐지한다.
- [ ] 원장 행 사이의 잔액 연속성 불일치를 탐지한다.
- [ ] 지갑 대기 잔액과 고정 회수 대기액에서 지급액을 뺀 값의 불일치를 탐지한다.
- [ ] Lot의 구매 원장 누락·금액 불일치와 Lot 상태·Recovery 불일치를 탐지한다.
- [ ] Recovery·대기 지급과 대응 CREDIT 원장의 누락·금액 불일치를 탐지한다.
- [ ] 뒤의 정상 검사 뒤에도 이전 불일치 결과가 변경되지 않고 남는다.
- [ ] 동시 스케줄 실행은 PostgreSQL advisory lock으로 하나만 수행한다.
- [ ] 집중 테스트와 전체 회귀 테스트가 통과한다.
