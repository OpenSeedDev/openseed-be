# VS-040 7필드 랭킹순 메인 API

## 사용자 결과

Guest를 포함한 사용자는 공개 범위와 관계없이 현재 게시된 랭킹을 순위대로 조회하고, 카드에 승인된 7개 항목만 확인한다.

## 근거와 선행 조건

- 백로그: VS-040, 선행 VS-039, resource lock `ranking`
- 긴급 배포용 사용자 override: 아직 Merge되지 않은 VS-039 PR #77의 Head `563a9141987bff1766943c43fa2f54a22648f115`를 선행 구현 증거로 사용한다.
- 병합 순서: PR #77이 먼저 Merge된 뒤 이 PR을 Merge한다.
- 통합 기획서 13.2와 통합 기능 명세는 랭킹순과 승인된 7개 카드 필드만 요구한다.

## 포함 범위

- 인증 없이 `GET /api/v1/rankings`로 현재 랭킹 전체를 순위 오름차순 조회
- 순위, 순위 변동, 제목, 한 줄 요약, 카테고리, 기업 관심 수, 좋아요 수 반환
- 새 진입 아이디어의 순위 변동은 `null`, 기존 아이디어는 `previousRankPosition - rankPosition`으로 반환
- `PUBLIC`, `SEMI_PUBLIC`, `MATCHING`을 구분하지 않고 같은 공개 카드 필드 반환
- 게시 시점의 기업 관심 수와 좋아요 수를 현재 랭킹 스냅샷에 함께 저장
- OpenAPI 계약 갱신

## 제외 범위

- 정렬 변경, 카테고리·공개 범위 필터, 복합 검색과 페이지네이션
- 현재가, 활성 보유 원금·보유자, 조회수, 계산·업데이트 시각, 점수 구성
- 아이디어 상세 공개 범위 정책 변경
- 기업 관심 등록·취소 API 자체의 변경

## API와 데이터 영향

- 응답은 JSON 배열이며 각 항목은 `rank`, `rankChange`, `title`, `summary`, `category`, `companyInterestCount`, `likeCount`만 가진다.
- DB 테이블과 제약은 변경하지 않는다. `ranking_current.components` JSON에 원시 기업 관심 수와 좋아요 수를 추가한다.
- 기존 게시 데이터에 원시 수가 없으면 0으로 호환 조회하며 다음 시간별 게시부터 실제 스냅샷 값을 사용한다.

## 인수 조건과 테스트 목록

- [ ] 현재 랭킹이 `rank_position` 오름차순으로 반환된다.
- [ ] 응답 항목에는 승인된 7개 필드만 존재한다.
- [ ] 이전 순위가 있으면 상승은 양수, 하락은 음수, 유지 0이며 새 진입은 `null`이다.
- [ ] 세 공개 범위가 인증 없이 동일한 카드 필드로 반환된다.
- [ ] Draft·Archived와 현재 랭킹에 없는 아이디어는 반환하지 않는다.
- [ ] 필터·정렬 Query Parameter와 상세 지표를 제공하지 않는다.
- [ ] 좋아요·기업 관심 원시 수가 랭킹 게시 스냅샷과 API에 보존된다.
- [ ] OpenAPI에 인증 없는 GET 계약과 200 응답이 게시된다.
