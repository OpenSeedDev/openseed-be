# VS-038 1시간 랭킹 계산

## 사용자 결과

게시된 아이디어의 시장 반응 신호를 매 UTC 정시 기준으로 같은 공식에 넣으면, 점수 구성 요소와 안정적인 순위가 항상 같은 결과로 계산된다.

## 근거와 선행 조건

- 백로그: VS-038, 선행 VS-029·VS-032, resource lock `ranking`, `metrics`
- 실제 Merge 증거: VS-029 PR #54와 VS-032 PR #50이 GitHub에서 Merge됨
- 통합 명세 RANK-01~04와 기획서 13.1의 승인 공식을 기준으로 한다.
- VS-029는 현재·시간별 조회 지표를, VS-032는 활성 Seed Unit Lot과 원금을 제공한다.

## 포함 범위

- 활성 원금·고유 활성 보유자·기업 관심·피드백·채택·좋아요·조회·정규화된 24시간 성장 신호의 점수화
- 명세의 가중치와 상한 적용
- 게시 후 7일 유예 뒤 하루마다 기준 점수의 2% 선형 감쇠(최대 기준 점수)
- 총점 동점 시 고유 활성 보유자, 채택 피드백, 기업 관심, 먼저 게시된 시각 순 정렬
- 완전 동일한 정책 키에는 아이디어 ID 오름차순을 마지막 안정화 키로 사용
- UTC 정시 계산 시각과 1부터 시작하는 순위가 포함된 불변 결과 생성

## 제외 범위

- DB에서 신호를 집계하는 Query와 RankingCurrent 저장·교체: VS-039
- 중복 Job, 분산 Lock, Scheduler와 원자적 게시: VS-039
- 랭킹 목록 API와 순위 변동: VS-040
- 기업 관심 등록·취소: VS-044
- 성장 신호 정규화 방식: 집계 경계에서 확정하며 이 슬라이스는 승인된 정규화 결과를 입력받는다.

## API와 데이터 영향

- 외부 HTTP API 변경 없음
- DB·Migration·ERD 변경 없음
- `ranking.calculate` 패키지에 입력 신호, 점수 결과, 순위 결과와 순수 계산기를 추가한다.

## 인수 조건과 테스트 목록

- [ ] 투자·다양성·기업·피드백·반응·성장 점수가 공식과 일치한다.
- [ ] 다양성 20, 기업 36, 피드백 30, 반응 20, 성장 15 상한을 넘지 않는다.
- [ ] 게시 후 정확히 7일까지 감쇠가 없고 8일째부터 기준 점수의 하루 2%를 감쇠한다.
- [ ] 총점 동점은 고유 활성 보유자·채택 피드백·기업 관심·게시 시각 순으로 결정한다.
- [ ] 완전히 같은 동점 키도 아이디어 ID로 안정적인 결과를 만든다.
- [ ] 음수·비정상 신호와 UTC 정시가 아닌 계산 시각을 거부한다.
- [ ] 같은 Fixture와 계산 시각은 같은 점수와 순위를 반환한다.

## 알려진 제한

이 슬라이스는 계산 코어만 제공한다. 실제 현재·시간별 지표와 Lot·피드백·좋아요·기업 관심을 한 시점으로 집계하고 결과를 게시하는 책임은 VS-039에 있다.
