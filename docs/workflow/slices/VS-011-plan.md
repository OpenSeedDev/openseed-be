# VS-011 공개 범위별 아이디어 상세 조회

## 사용자 결과

Guest, 로그인 User, 작성자, Company가 게시된 아이디어 상세를 조회할 때 공개형·반공개형·매칭형 정책에 맞는 필드만 받으며, 숨겨야 할 사업 상세와 검증 질문은 JSON 응답에 존재하지 않는다.

## 근거와 선행 조건

- 백로그: VS-011, 선행 VS-010, resource lock `idea`·`security`
- 실제 Merge 증거: VS-010 PR #48이 `72ab1b6fc8c0148bb09493864d280d709914b80b`로 2026-07-21T10:12:25Z Merge됨
- 기능 명세 VIS-01~04와 DETAIL-01, 공개 범위 API 강제 정책을 적용한다.

## 포함 범위

- `GET /api/v1/ideas/{ideaId}`를 Guest도 호출 가능한 상세 조회 API로 확장
- 공개형: Guest를 포함한 모든 조회자에게 전체 내용과 검증 질문 공개
- 반공개형: Guest에게 제목·카테고리·요약·문제 정의와 공통 게시 정보만 공개하고 로그인 User·Company·작성자에게 전체 내용과 검증 질문 공개
- 매칭형: Guest·User·Company에게 요약과 공통 게시 정보만 공개하고 작성자에게만 전체 내용과 검증 질문 공개
- Draft는 기존과 같이 작성자만 조회하며 다른 조회자에게 존재 여부를 숨긴다.
- Authorization 헤더가 없으면 Guest, 헤더가 있으면 유효한 활성 세션만 허용한다.
- 누출 방지를 위해 비공개 필드는 `null`이 아니라 JSON 응답 자체에서 제외한다.
- OpenAPI에 선택적 Bearer 인증과 공개 범위별 응답 계약을 반영한다.

## 제외 범위

- 비공개 검증형, 상세 열람 요청·승인
- 조회수와 지표, 피드백, 타임라인, 기업 관심, Unit 구매
- 아이디어 수정·보관과 사용자용 버전 이력
- 목록·검색의 공개 카드 응답

## API와 데이터 영향

- 기존 URI를 유지하면서 게시 상세 응답에 `visibility`, `currentUnitPrice`, `publishedAt`을 추가한다.
- 공통 필드는 ID·상태·제목·카테고리·요약·공개 범위·현재가·게시 시각·생성·수정 시각이다.
- 상세 필드는 문제·대상 고객·해결책·수익 모델·검증 질문이며 Matrix에 따라 응답에서 생략한다.
- 새 테이블이나 컬럼이 필요하지 않아 Migration과 ERD 구조 변경은 없다.
- 존재하지 않는 아이디어와 권한 없는 Draft는 모두 `404 IDEA_NOT_FOUND`다.
- 위조·만료·폐기된 선택적 Bearer 토큰은 `401 INVALID_ACCESS_TOKEN`이다.

## 인수 조건과 테스트 목록

- [ ] 공개형은 Guest·User·Company에게 전체 상세와 검증 질문을 반환한다.
- [ ] 반공개형 Guest 응답은 문제 정의를 포함하고 대상 고객·해결책·수익 모델·검증 질문 필드를 포함하지 않는다.
- [ ] 반공개형 로그인 User·Company는 전체 상세를 조회한다.
- [ ] 매칭형 Guest·User·Company 응답은 요약과 공통 게시 정보만 포함하고 문제·대상 고객·해결책·수익 모델·검증 질문 필드를 포함하지 않는다.
- [ ] 매칭형 작성자는 전체 상세를 조회한다.
- [ ] Draft는 작성자만 전체 조회하고 Guest·다른 사용자는 동일한 404를 받는다.
- [ ] 선택적 Authorization이 위조되면 Guest로 강등하지 않고 401을 반환한다.
- [ ] 응답에 작성자 ID·이메일, 세션·토큰, 내부 버전이 노출되지 않는다.
- [ ] OpenAPI에 200·401·404와 선택적 Bearer 인증·공개 범위 설명이 노출된다.
- [ ] 집중 테스트와 공개 범위 고위험 전체 테스트가 통과한다.
