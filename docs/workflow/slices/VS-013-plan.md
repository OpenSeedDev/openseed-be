# VS-013 검증 질문 관리

## 사용자 결과

아이디어 작성자는 검증 질문 1~3개를 한 번에 저장하고, 목록 순서를 바꾸거나 문구를 수정한 뒤 아이디어 상세 조회에서 저장된 순서대로 확인할 수 있다.

## 근거와 선행 조건

- 백로그: VS-013, 선행 VS-009, resource lock `idea`
- 선행 PR #29가 2026-07-21에 Merge됐고 Merge 커밋 `087d640d87b5283e7ae952e7ad00bc63a5091344`이 현재 기준 커밋이다.
- 통합 기능 명세 IDEA-04는 사용자가 검증 질문 1~3개를 선택하거나 직접 입력하도록 요구한다.
- 데이터 명세는 `ValidationQuestion(id, ideaId, question, sortOrder)`를 정의한다.

## 범위

### 포함

- `PUT /api/v1/ideas/{ideaId}/validation-questions`로 전체 질문 목록 교체
- 질문 1~3개 등록, 문구 수정, 배열 순서에 따른 재정렬
- 작성자 소유권과 활성 Bearer 세션 검증
- `GET /api/v1/ideas/{ideaId}` 상세 응답에 질문 ID, 문구, 1부터 시작하는 순서 표시
- PostgreSQL Migration, JPA 매핑, OpenAPI와 ERD 갱신

### 제외

- AI 추천 질문 생성과 질문 템플릿 카탈로그
- 게시 가능 여부 검증과 게시 처리
- 질문별 답변, 피드백, 통계
- 질문 추가·삭제·이동을 위한 개별 부분 수정 API
- 비작성자·Guest·Company 상세 공개 정책

## API와 데이터 계약

- 저장 요청은 `questions` 배열 하나를 받으며 원소는 질문 문자열이다.
- 배열은 최소 1개, 최대 3개여야 하고 각 질문은 공백일 수 없다.
- 질문 앞뒤 공백은 제거한다. 명세에 길이 상한이 없으므로 DB에는 `TEXT`로 저장하고 임의 상한을 추가하지 않는다.
- 요청 배열의 순서를 `sortOrder` 1~3으로 저장하고 응답도 같은 순서로 반환한다.
- 전체 교체 API이므로 재호출하면 기존 질문을 요청 목록으로 원자적으로 대체한다.
- 다른 작성자 또는 존재하지 않는 아이디어는 존재를 숨기기 위해 동일한 `404 IDEA_NOT_FOUND`로 응답한다.
- 인증 누락·위조·폐기 세션은 `401 INVALID_ACCESS_TOKEN`으로 응답한다.

## 인수 조건과 테스트 목록

- 작성자가 질문 1~3개를 저장하면 `201`이 아닌 `200 OK`로 정렬된 전체 목록을 받는다.
- 문구와 배열 순서를 바꿔 다시 저장하면 이전 목록이 원자적으로 교체되고 상세 조회에 새 순서가 보인다.
- 질문 문자열의 앞뒤 공백은 제거된다.
- 0개, 4개, 공백 질문, `null` 원소는 `400 VALIDATION_ERROR`다.
- 인증 없는 요청은 `401 INVALID_ACCESS_TOKEN`이다.
- 다른 사용자는 질문을 변경할 수 없고 `404 IDEA_NOT_FOUND`를 받는다.
- 질문 저장 중 오류가 나면 기존 목록이 부분 삭제되지 않는다.
- 응답에는 작성자 이메일, 인증 토큰, 세션 ID가 없다.
- OpenAPI에 PUT·Bearer 인증·200/400/401/404와 질문 배열 1~3개 계약이 노출된다.
- PostgreSQL Testcontainers에서 새 Migration과 `ddl-auto: validate`가 통과한다.
- 집중 테스트와 `./gradlew clean test` 전체 회귀 테스트가 통과한다.
