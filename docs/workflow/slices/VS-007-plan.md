# VS-007 일회성 회사 인증 메일 발송

## 사용자 결과

회사 프로필을 등록한 로그인 사용자는 인증 메일 발송을 요청할 수 있다. 서버는 30분 동안 유효한 고강도 일회성 토큰을 만들고 원문은 메일 링크에만 전달하며, DB에는 SHA-256 해시만 저장한다. 메일은 요청 트랜잭션 커밋 뒤 비동기로 발송된다.

## 선행 조건

- VS-006 PR #28이 `2026-07-21T05:27:24Z`에 Merge되었고 Merge 커밋 `a40793533cff7e72510202f03439985e46c50485`가 기준 커밋 `0a69a3c08a090f890ce952704c07f26d2a9e4cc2`의 조상이다.
- 백로그 선행 조건은 VS-006, resource lock은 `company`, `mail`이다.

## 포함 범위

- `POST /api/v1/companies/verifications`
- Bearer Access Token과 활성 세션 검증
- 회사 프로필 소유자만 발송 요청
- URL-safe 256-bit 임의 토큰, SHA-256 해시 저장, 기본 30분 만료
- 재발송 시 기존 미사용 토큰 무효화
- 트랜잭션 커밋 뒤 비동기 SMTP 메일 발송
- 메일 Provider 오류를 요청 응답과 분리하고 민감정보 없는 로그만 기록
- CompanyVerification Migration, ERD, OpenAPI 계약

## 제외 범위

- 토큰 확인, 만료·위조·재사용 판정과 Company 권한 부여: VS-008
- 관리자 승인, 일반 사용자 이메일 인증
- 메일 전송 상태 조회 API, 영속 Outbox, 운영 지표: 후속 운영 슬라이스

## API·데이터 영향

- 요청 본문 없음. 성공은 `202 Accepted`와 `expiresAt`만 반환한다.
- 회사 프로필이 없으면 `409 COMPANY_PROFILE_REQUIRED`, 이미 인증된 프로필은 `409 COMPANY_ALREADY_VERIFIED`, 인증 실패는 `401 INVALID_ACCESS_TOKEN`이다.
- `company_verifications`에는 프로필, 토큰 해시, 만료·사용·무효화·생성 시각만 저장한다. 토큰 원문과 회사 이메일은 저장하지 않는다.
- 확인 URL, 발신자, SMTP 연결 정보와 만료 시간은 외부 설정으로 주입한다.

## 인수 조건과 테스트 목록

- 유효한 프로필의 요청이 `202`를 반환하고 64자 소문자 16진 SHA-256 해시와 30분 만료를 저장한다.
- 저장 토큰 해시가 메일 링크 원문 토큰의 SHA-256과 일치하고 원문은 DB·API 응답에 없다.
- 메일 발송은 HTTP 요청 처리 스레드가 아닌 별도 비동기 스레드에서 트랜잭션 커밋 후 수행된다.
- 재발송은 앞선 미사용 인증을 무효화하고 새 토큰 하나만 활성 상태로 둔다.
- 회사 프로필 없음, 이미 인증됨, 인증 누락·위조·폐기 세션을 거부한다.
- SMTP 실패가 API 응답을 되돌리지 않고 비밀값을 애플리케이션 로그에 직접 기록하지 않는다.
- OpenAPI에 Bearer 보안, `202`, `401`, `409`와 민감 필드 없는 응답 계약이 노출된다.
- PostgreSQL Testcontainers에서 새 Migration과 `ddl-auto: validate`가 통과한다.
