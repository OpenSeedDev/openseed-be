# VS-029 상세 조회수 집계

## 사용자 결과

사용자가 게시된 아이디어 상세을 열면 중복 제거된 조회수가 응답에 표시되고, 랭킹 계산이 사용할 현재 누계와 UTC 시간별 증가량이 함께 저장된다.

## 근거와 선행 조건

- 백로그: VS-029, 선행 VS-011, resource lock `metrics`
- 실제 Merge 증거: VS-011 PR #49가 `482c811d2307a2224f63a6b55b95bab208ce88c2`로 2026-07-21T10:49:14Z Merge됨
- 기획 REACT-02는 상세 조회 집계와 중복·비정상 반복 제한을 요구한다.
- 데이터 명세는 `ViewEvent`, `IdeaMetricCurrent`, `IdeaMetricHourly`를 정의한다.

## 포함 범위

- 기존 `GET /api/v1/ideas/{ideaId}` 성공 응답에 `viewCount` 추가
- 게시된 아이디어의 조회 성공만 집계
- 로그인 사용자는 사용자 ID, Guest는 서버 발급 HTTP 세션 ID로 조회자를 구분
- 같은 아이디어·조회자·UTC 시간 버킷당 한 번만 조회 인정
- 원본 `idea_view_events`, 현재 누계 `idea_metric_current`, 시간별 증가량 `idea_metric_hourly` 원자 갱신
- DB unique 제약과 upsert로 동시 중복 조회·유실 갱신 방지
- OpenAPI, ERD, 새 UTC Flyway Migration 갱신

## 제외 범위

- 좋아요·피드백·기업 관심·Unit 지표: 각 후속 슬라이스
- 랭킹 점수 계산과 게시: VS-038·VS-039
- 운영자용 원본 조회·어뷰징 관리 API
- IP·User-Agent·기기 지문 수집

## 중복 제거 정책

- 로그인 조회자는 재로그인해도 같은 사용자 ID를 사용한다.
- Guest는 서버 HTTP 세션 ID를 사용하며 IP나 User-Agent를 저장하지 않는다.
- 시간 버킷은 `Instant`를 UTC 정시로 내림한 값이다.
- 동일 조회자가 같은 시간 버킷에 반복 조회하면 이벤트·현재 누계·시간별 증가량을 변경하지 않는다.
- 다음 UTC 시간 버킷의 첫 조회는 새 조회로 인정한다.

## API와 데이터 계약

- 공개 범위에 따라 허용된 게시 상세만 집계하고 404·401 응답과 Draft 작성자 조회는 집계하지 않는다.
- `viewCount`는 공개 범위와 관계없이 게시 상세의 공통 공개 지표다.
- 최초 조회는 누계 1과 시간별 증가량 1을 생성한다.
- 동시 서로 다른 조회자는 누계와 증가량을 잃지 않고 모두 반영한다.
- 집계 실패 시 상세 응답과 집계 데이터가 같은 트랜잭션에서 롤백된다.

## 인수 조건과 테스트 목록

- [ ] Guest의 게시 상세 첫 조회가 `viewCount=1`과 현재·시간별 지표를 만든다.
- [ ] 같은 Guest 세션의 같은 시간 반복 조회는 한 번만 집계한다.
- [ ] 로그인 사용자의 복수 인증 세션은 같은 시간에 사용자당 한 번만 집계한다.
- [ ] 다음 UTC 시간의 첫 조회는 누계와 새 시간 버킷을 증가시킨다.
- [ ] 서로 다른 Guest·User 조회자는 각각 집계한다.
- [ ] 동시 중복 조회는 한 번, 동시 고유 조회는 모두 반영한다.
- [ ] Draft·숨겨진 아이디어·잘못된 인증·없는 아이디어는 집계하지 않는다.
- [ ] 세 공개 범위의 허용된 상세 응답에 동일하게 `viewCount`가 표시된다.
- [ ] OpenAPI, Migration, ERD와 PostgreSQL `ddl-auto: validate`가 일치한다.
- [ ] 집중 테스트와 고위험 전체 테스트가 통과한다.

## 미결정 사항

분산 서버 간 Guest 세션 공유와 장기 기기 지문은 MVP 운영 범위 밖이며, 현재는 서버 세션 쿠키 단위로 처리한다.
