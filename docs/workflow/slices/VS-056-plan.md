# VS-056 일일 첫 접속 100P

## 사용자 결과

활성 사용자는 Asia/Seoul 기준 하루의 첫 로그인에 100P를 받고, 같은 날 다시 로그인해도 중복 지급되지 않는다.

## 근거와 선행 조건

- 백로그: VS-056, 선행 VS-030, `wallet`·`auth` resource lock
- 실제 Merge 증거: VS-030 PR #30이 `9454302d4cd93f7df13a03f5737dd98d452c4b6c`로 2026-07-21T06:28:10Z Merge됨
- 활동 Point 정책 선행 구현: VS-031 PR #43이 `e4d247c7b1427e2f89515682f2eeebbb1346bc16`로 2026-07-21T08:10:44Z Merge됨
- VS-031은 사용자 지갑 잠금, 하루 300P 활동 한도, 지갑 2,000P 상한, 초과 소멸과 출처 멱등 원장을 제공한다.

## 포함 범위

- 성공한 로그인에서 당일 첫 접속 100P 지급
- Asia/Seoul 정책 날짜별 사용자당 한 번만 지급
- 연속·동시 로그인 중복 지급 방지
- 일일 활동 한도와 지갑 상한 적용 및 초과분 원장 기록
- 로그인 세션 생성과 Point 지급을 하나의 트랜잭션으로 처리
- Point 원장 출처 유형과 DB 제약, ERD 갱신

## 제외 범위

- 로그인 외 보호 API 호출을 별도의 접속으로 집계
- 연속 출석, 출석 달력, 알림, 운영자 수동 지급
- 로그인 응답에 Point 지급 결과 추가
- Point 차감·Unit 구매·회수

## 애플리케이션과 데이터 영향

- 로그인 성공 시 `PointRewardService`가 `DAILY_FIRST_ACCESS` 보상을 지급한다.
- 사용자 UUID와 Asia/Seoul 정책 날짜로 결정적인 출처 UUID를 만들어 같은 날 재호출을 멱등 처리한다.
- `point_ledgers.source_type`에 `DAILY_FIRST_ACCESS`를 추가하는 새 Flyway migration을 적용한다.
- 공개 API 요청·응답은 바뀌지 않으므로 OpenAPI 경로 변경은 없다.

## 인수 조건과 테스트 목록

- [ ] 당일 첫 성공 로그인은 100P를 지급하고 원장에 정책 날짜를 기록한다.
- [ ] 같은 사용자의 같은 날 재로그인은 새 Point나 원장을 만들지 않는다.
- [ ] Asia/Seoul 자정 이후 첫 로그인은 다음 날 보상을 새로 지급한다.
- [ ] 잘못된 자격 증명과 정지 계정은 보상을 지급하지 않는다.
- [ ] 당일 활동 지급액이 250P이면 첫 접속 보상 중 50P만 지급하고 50P는 소멸 기록한다.
- [ ] 동시 로그인도 사용자·정책 날짜당 한 번만 보상한다.
- [ ] 인증 세션 저장 또는 Point 지급 실패 시 로그인 성공 응답과 부분 데이터가 남지 않는다.
- [ ] 집중 테스트와 고위험 전체 테스트가 통과한다.
