# VS-012 아이디어 수정과 내부 버전 저장

## 사용자 결과

게시자는 게시된 아이디어의 내용을 직접 수정할 수 있다. 수정된 최신 내용과 마지막 업데이트 시각은 상세에 반영되고, 내부에는 연속된 버전 번호와 전체 스냅샷이 변경 이유 없이 보존된다.

## 근거와 선행 조건

- 백로그: VS-012, 선행 VS-010, resource lock `idea`
- 실제 Merge 증거: VS-010 PR #48이 `72ab1b6fc8c0148bb09493864d280d709914b80b`로 2026-07-21T10:12:25Z Merge됨
- VS-010은 게시 시 버전 1 전체 스냅샷을 생성한다.
- VS-011은 최신 아이디어 상세을 공개 범위에 맞게 노출하고 `updatedAt`을 제공한다.

## 포함 범위

- `PATCH /api/v1/ideas/{ideaId}`
- 작성자의 게시된 아이디어 제목·카테고리·요약·문제·대상 고객·해결책·수익 모델 수정
- 아이디어 행 잠금으로 동시 수정 직렬화
- 수정 내용, 현재 공개 범위, 현재 검증 질문, 수정자 ID, 수정 시각을 다음 번호의 `IdeaVersion`으로 원자 저장
- `ideas.updated_at`을 동일한 애플리케이션 시각으로 갱신
- `idea_versions`의 UPDATE·DELETE를 거부하는 append-only DB 제약
- OpenAPI, ERD, 새 UTC Flyway Migration 갱신

## 제외 범위

- 공개 범위·상태·가격·게시 시각 변경
- 검증 질문 수정: VS-013 API 사용
- 변경 이유·수정 방향·변경 필드·피드백 연결
- 사용자용 버전 목록·상세·비교·복원 API
- 수정 타임라인 조회: VS-017

## API와 데이터 계약

- 요청은 완성된 7개 내용 필드를 모두 받으며 Draft 생성과 동일한 최대 길이를 적용한다.
- 존재하지 않거나 다른 작성자의 아이디어는 `404 IDEA_NOT_FOUND`로 동일 응답한다.
- 게시되지 않은 아이디어는 `409 IDEA_NOT_PUBLISHED`로 거부한다.
- 응답은 최신 내용과 `updatedAt`을 반환하지만 버전 번호·내부 이력은 반환하지 않는다.
- 버전 번호는 기존 최대 번호 다음 값이며 아이디어 행 잠금으로 동시 요청에도 중복되지 않는다.
- 아이디어 변경과 버전 저장은 한 트랜잭션으로 완료되거나 함께 롤백한다.

## 인수 조건과 테스트 목록

- [ ] 작성자는 게시된 아이디어 내용을 수정하고 최신 내용·`updatedAt`을 받는다.
- [ ] 버전 2는 수정된 전체 내용·기존 공개 범위·현재 검증 질문·작성자 ID·동일 수정 시각을 보존한다.
- [ ] 연속 수정은 2, 3 순서의 불변 스냅샷을 남긴다.
- [ ] 비작성자·비인증·존재하지 않는 아이디어는 수정할 수 없다.
- [ ] Draft는 게시된 아이디어 수정 API로 수정할 수 없다.
- [ ] 유효하지 않은 필드와 길이는 `400 VALIDATION_ERROR`다.
- [ ] 동시 수정에도 버전 번호가 중복되지 않고 최신 아이디어와 각 스냅샷이 일관된다.
- [ ] 버전 저장 실패 시 아이디어 최신 내용과 `updatedAt`도 롤백된다.
- [ ] DB에서 기존 버전 UPDATE·DELETE가 거부된다.
- [ ] OpenAPI에 PATCH 계약만 추가되고 버전 조회 경로는 없다.
- [ ] 집중 테스트와 고위험 전체 테스트가 통과한다.
