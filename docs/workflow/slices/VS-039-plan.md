# VS-039 원자적 랭킹 게시

## 사용자 결과

매 UTC 정시에 같은 시간의 랭킹은 한 번만 계산되며, 조회자는 계산 도중의 일부 결과가 아니라 완전히 교체된 현재 랭킹만 보게 된다.

## 근거와 선행 조건

- 백로그: VS-039, 선행 VS-038, resource lock `ranking`
- 실제 Merge 증거: VS-038 PR #71이 `12e022d4ae922f696ac26b5248c669abdc1b99f8`로 Merge됨
- 통합 명세 RANK-01~04와 6.4는 Current·Hourly 지표로 시간별 점수를 한 번 계산하고 완성된 `RankingCurrent`를 원자 게시하도록 요구한다.

## 포함 범위

- UTC 직전 정시를 대상으로 매시간 실행되는 내부 Scheduler
- 게시된 아이디어의 활성 Lot 원금·고유 보유자, 활성 피드백·채택, 좋아요, 누적 조회와 최근 24시간 조회 성장 신호 집계
- 최근 24시간 조회 증가량을 같은 실행의 최댓값 대비 0~15로 정규화해 VS-038 계산기에 전달
- 전역 게시 잠금과 시간별 실행 유일 제약으로 중복·동시 Job 방지
- 기존 순위를 `previousRankPosition`으로 보존하며 `ranking_current` 전체를 한 트랜잭션에서 교체
- 실패 시 실행 기록과 현재 랭킹을 함께 롤백해 같은 시간을 안전하게 재시도
- 새 UTC Flyway Migration과 ERD 갱신

## 제외 범위

- 랭킹 목록 HTTP API와 순위 변동 표시: VS-040
- 일일 마감 점수와 Unit 가격 갱신: VS-042
- 이미 성공한 시간의 운영자 수동 재실행: VS-043
- 기업 관심 집계: VS-044가 선행 관계가 아니므로 현재 0이며 해당 기능에서 집계 쿼리를 확장한다.

## API와 데이터 영향

- 외부 HTTP API 변경 없음, OpenAPI 변경 없음
- `ranking_publish_lock`: 전체 게시 직렬화를 위한 단일 잠금 행
- `ranking_runs`: UTC 대상 시간별 성공 실행 한 건
- `ranking_current`: 아이디어별 현재·직전 순위, 총점·구성 점수, 계산 시각
- Scheduler는 현재 시각을 UTC 정시로 내림해 실행하며 같은 대상 시간 재호출은 변경 없이 종료한다.

## 인수 조건과 테스트 목록

- [ ] 게시된 아이디어의 현재 지표와 활성 Lot·피드백·좋아요·24시간 조회를 집계해 VS-038 순위를 게시한다.
- [ ] 한 대상 시간을 순차 또는 동시에 실행해도 성공 실행과 현재 결과는 한 벌만 존재한다.
- [ ] 새 게시에서 직전 순위를 보존하고 더 이상 게시 대상이 아닌 아이디어는 현재 랭킹에서 제거한다.
- [ ] 결과 쓰기 중 제약 위반이 발생하면 기존 현재 랭킹과 실행 기록이 모두 유지·롤백된다.
- [ ] 더 오래된 시간은 최신 게시 결과를 덮어쓰지 않는다.
- [ ] UTC 정시가 아닌 대상 시간을 거부한다.
- [ ] Migration, JPA validate, 집중 동시성 테스트와 전체 회귀 테스트가 통과한다.

## 알려진 제한

현재 명세에는 24시간 성장 정규화의 별도 공식이 없어 배치 내 최대 조회 증가량 대비 선형 0~15를 사용한다. 기업 관심은 VS-044가 Merge된 뒤 집계에 연결한다.
