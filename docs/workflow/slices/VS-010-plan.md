# VS-010 아이디어 게시

## 사용자 결과

작성자는 완성한 Draft와 검증 질문을 세 가지 공개 범위 중 하나로 게시한다. 게시와 동시에 최초 버전, 10P 초기 가격, 게시 타임라인과 해당되는 Point 보상이 원자적으로 생성된다.

## 근거와 선행 조건

- 백로그: VS-010, 선행 VS-009, resource lock `idea`·`wallet`
- 실제 Merge 증거: VS-009 PR #29가 `087d640d87b5283e7ae952e7ad00bc63a5091344`로 2026-07-21T04:03:16Z Merge됨
- VS-009는 작성자 전용 Draft 생성·조회, VS-013은 검증 질문 1~3개 관리를 제공한다.
- VS-031은 일일 활동 한도·지갑 상한·Append-only 원장을 적용하는 `PointRewardService`를 제공한다.

## 포함 범위

- `POST /api/v1/ideas/{ideaId}/publish`
- `PUBLIC`, `SEMI_PUBLIC`, `MATCHING` 공개 범위 확정과 `PUBLISHED` 전환
- 제목·카테고리·요약·문제·대상 고객·해결책·수익 모델 및 검증 질문 1~3개 완성 검증
- 최초 `IdeaVersion` 전체 스냅샷, 10P 현재 Unit 가격, `PUBLISHED` 타임라인
- 공개형·반공개형 게시 50P를 Asia/Seoul 하루 2회까지 지급
- 아이디어 행 잠금으로 중복·동시 게시 방지
- 새 UTC Flyway Migration, JPA 매핑, OpenAPI와 ERD 갱신

## 제외 범위

- Guest/User/Author/Company별 상세 필드 노출: VS-011
- 게시 후 수정과 후속 버전: VS-012
- 타임라인 조회: VS-017
- Unit 구매·가격 변경: VS-032 이후
- 매칭형 게시 보상: 기획의 공개·반공개 보상 대상에서 제외

## API와 데이터 영향

- 요청은 `visibility` 하나만 받으며 지원하지 않는 값은 `400 VALIDATION_ERROR`다.
- 존재하지 않거나 다른 작성자의 아이디어는 `404 IDEA_NOT_FOUND`로 동일 응답한다.
- 이미 게시된 아이디어는 `409 IDEA_ALREADY_PUBLISHED`다.
- 응답은 아이디어 ID, 상태, 공개 범위, 10P 현재가, 게시 시각과 게시 보상 원래·실지급·소멸 금액을 반환한다.
- 매칭형은 게시 보상 금액을 모두 0으로 반환하고 보상 원장을 생성하지 않는다.
- 공개형·반공개형의 세 번째 당일 게시는 허용하되 50P를 전액 소멸한 원장을 남긴다.

## 인수 조건과 테스트 목록

- [ ] 작성자는 세 공개 범위 각각으로 완성된 Draft를 게시할 수 있다.
- [ ] 게시 시 상태·공개 범위·10P 가격·게시 시각이 저장된다.
- [ ] 최초 버전은 번호 1과 전체 내용·공개 범위·수정자·시각을 보존한다.
- [ ] `PUBLISHED` 타임라인이 한 건 생성된다.
- [ ] 공개형·반공개형은 50P를 지급하고 매칭형은 지급하지 않는다.
- [ ] 공개형·반공개형 게시 보상은 하루 2회까지만 지급되고 세 번째는 전액 소멸한다.
- [ ] 필수 내용이나 검증 질문 1~3개가 없으면 게시되지 않고 부수 데이터도 생성되지 않는다.
- [ ] 비작성자·비인증 요청과 이미 게시된 Draft를 거부한다.
- [ ] 동시 게시 요청에도 버전·타임라인·보상은 한 번만 생성된다.
- [ ] OpenAPI, 새 Migration, ERD, PostgreSQL `ddl-auto: validate`가 일치한다.
- [ ] 집중 테스트와 고위험 전체 테스트가 통과한다.
