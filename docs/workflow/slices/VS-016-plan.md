# VS-016 아이디어 보관

## 사용자 결과

게시자는 더 이상 공개하지 않을 자신의 게시 아이디어를 보관할 수 있다. 보관된 아이디어는 작성자의 내 아이디어 목록과 상세에서는 유지되지만 다른 사용자에게는 노출되지 않는다.

## 근거와 전달 방식

- 백로그: VS-016, 선행 VS-010, resource lock `idea`
- Fast Build stack parent: VS-012 PR #52, `codex/vs-012-update-history`, Head `180bbee85b2f1efaf322606b86a904afc8ea8007`
- Stack depth 2이며 병합 순서는 VS-012 다음 VS-016이다.
- Merge되지 않은 parent를 기반으로 하므로 parent 변경 시 집중 테스트와 diff를 다시 검증한다.

## 포함 범위

- `POST /api/v1/ideas/{ideaId}/archive`
- 작성자 인증 및 작성자 소유 아이디어 행 잠금
- `PUBLISHED` 아이디어를 `ARCHIVED`로 전환하고 `updatedAt` 갱신
- 보관 후 비작성자 상세 조회 차단과 작성자 상세·목록 유지
- 동시 보관 요청 직렬화
- OpenAPI 계약 갱신

## 제외 범위

- Draft 삭제 또는 보관
- 보관 취소·재게시
- 버전 스냅샷과 타임라인 조회
- Unit 보유·가격·랭킹·기업 관심의 후속 정리 정책

## API와 오류 계약

- 정상 응답은 ID, `ARCHIVED` 상태와 `updatedAt`을 반환한다.
- 비인증 요청은 `401 INVALID_ACCESS_TOKEN`이다.
- 존재하지 않거나 다른 작성자의 아이디어는 `404 IDEA_NOT_FOUND`로 동일하게 처리한다.
- Draft 또는 이미 보관된 아이디어는 `409 IDEA_NOT_PUBLISHED`다.

## 인수 조건과 테스트 목록

- [ ] 작성자는 게시된 아이디어를 보관하고 상태·업데이트 시각을 받는다.
- [ ] 보관 후 작성자는 상세와 내 목록에서 아이디어를 확인한다.
- [ ] 보관 후 Guest와 다른 사용자는 상세를 조회할 수 없다.
- [ ] 비인증·비작성자·존재하지 않는 아이디어를 거부한다.
- [ ] Draft와 이미 보관된 아이디어를 거부하고 상태를 바꾸지 않는다.
- [ ] 동시 보관은 한 요청만 성공하고 최종 상태가 일관된다.
- [ ] OpenAPI에 인증·성공·오류 응답 계약이 노출된다.
