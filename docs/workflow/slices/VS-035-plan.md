# VS-035 Seed Unit Lot 전체 회수

## 사용자 결과

로그인 사용자는 구매 후 24시간이 지난 자신의 Seed Unit Lot 전체를 회수하고, 회수 요청 시점 현재가로 고정된 실현액 중 정책 한도 내 금액은 즉시 지갑으로 받고 나머지는 회수 대기 잔액으로 보관할 수 있다.

## 근거와 선행 조건

- 백로그: VS-035, 선행 VS-032, resource lock `unit`·`wallet`
- 실제 Merge 증거: VS-032 PR #50, merge commit `535d258ea66545359f1ad501e57f9aca5545b5dd`
- 승인 정책: Lot 전체 회수, 정확히 24시간 잠금, 요청 시점 현재가, 일일 즉시 지급 500P, 지갑 상한 2,000P, 미지급 실현액 고정

## 포함 범위

- `POST /api/v1/unit-lots/{lotId}/recover`
- 소유자·활성 상태·24시간 잠금 검증
- 현재가 기준 전체 Unit 실현액 계산과 Lot 회수 상태 전환
- 일일 500P 및 지갑 2,000P 여유분 범위 내 즉시 지급
- 미지급액을 지갑의 회수 대기 잔액과 회수 기록에 고정
- 같은 Lot의 순차·동시 재요청 멱등 처리
- 새 UTC Flyway Migration, JPA 매핑, OpenAPI와 ERD 갱신

## 제외 범위

- 부분 회수
- 회수 대기 잔액 수동 지급: VS-036
- 회수 이력 목록 API
- 자동 지급·양도·현금화·외부 거래

## API와 데이터 영향

- 요청 본문 없이 Lot 전체를 회수한다. `units`를 지정한 요청은 부분 회수 시도로 거부한다.
- 없는 Lot과 타인 소유 Lot은 `404 UNIT_LOT_NOT_FOUND`, 잠금 미해제는 `409 UNIT_LOCKED`, 부분 회수는 `400 PARTIAL_RECOVERY_NOT_SUPPORTED`다.
- `seed_unit_recoveries`는 Lot별 하나의 고정된 회수 가격·실현액·즉시 지급액·대기액·회수 시각을 보존한다.
- 즉시 지급분은 `CREDIT/UNIT_RECOVERY` Point 원장으로 기록하고, 대기액은 소멸시키거나 재평가하지 않는다.

## 인수 조건과 테스트 목록

- [ ] 잠금 해제 직전에는 거부하고 정확한 해제 시각에는 성공한다.
- [ ] 소유자만 회수할 수 있고 타인의 Lot 존재를 노출하지 않는다.
- [ ] Lot 전체 Unit과 요청 시점 현재가로 실현액을 고정하고 상태를 `RECOVERED`로 바꾼다.
- [ ] 일일 500P와 지갑 2,000P 여유분만 즉시 지급하고 나머지를 대기 잔액으로 기록한다.
- [ ] 회수 뒤 가격이 바뀌어도 회수 기록과 대기 잔액은 바뀌지 않는다.
- [ ] 부분 회수 요청은 부작용 없이 거부한다.
- [ ] 같은 Lot의 순차·동시 재요청은 회수·지갑·원장을 중복 생성하지 않는다.
- [ ] 인증·오류·OpenAPI·DB 제약과 전체 회귀 테스트가 통과한다.
